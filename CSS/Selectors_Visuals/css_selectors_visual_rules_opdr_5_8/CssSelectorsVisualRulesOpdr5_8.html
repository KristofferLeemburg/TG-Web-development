Opdracht 6. Er zijn verschillende selectors, bijvoorbeeld;

  • child selectors
  • adjacent selectors
  • general selectors

Maak in een aparte html bestand voor elke selector een voorbeeld en maak een link in je antwoord naar dat html bestand. Geef in je antwoord ook aan wat de voordelen zijn van elk selector.

  • Child selector.
      • Child Selector: Child Selector wordt gebruikt om alle elementen te matchen die child zijn van een gespecificeerd element. Het geeft de relatie weer tussen twee elementen. Het element > element selector selecteert die elementen die zijn de children van een specifieke parent.

  • General selector.
      • General sibling is een weinig bekende maar zeer nuttige CSS-selector. Het maakt de styling mogelijk van een element dat een broer of zus is van een ander.

      Voorbeeld
      Syntax:
      h2 ~ p { color: red; }

      The general sibling selector is part of de family of combinator selectors in CSS and is identified with de tilde ~ character. It looks a bit like this: In that example, you would be selecting all paragraphs in een article that come after de featured image (een element met een class name of “featured-image”) and making them of slightly smaller font-size. This selects elements at de same hierarchy level. In this example .featured-image and de p elements zijn at de same hierarchy. If de selector continued past de p or before .featured-image, de normal rules apply. So .featured-image ~ p span still would select spans that zijn descendents of whatever .featured-image ~ p matches. The spec for selectors level 4 calls these

  • Adjacent selector.
      • Een adjacent sibling combinator selector geeft je de mogelijkheid om een element te selecteren die direct na een ander specifiek element komt. Deze selectors kunnen je helpen om je stijl toe te passen in een contextuele manier. Mischien dat je een aantal artikelen hebt op je website. En de meeste van deze artikelen starten met <p> elements en het ziet er goed uit. Maar sommige van deze artikelen starten met <ul>, welke een boven en onder een margin hebben om er goed uit te zien met een artikel. Maar wanneer je een artikel begint je een grote lege ruimte hebt. Dan zal de latere selector erboven de margin erboven weghalen wanneer je een nieuw artikel begint.
